# DocMind Agent 实施计划与当前进度

## 1. 实施目标

本项目当前的目标已经从“先把系统跑起来”转到“在可运行基线之上持续优化文档 Agent 质量”。因此本文件记录的是当前真实进度和下一阶段重点，而不是最初的空白立项计划。

## 2. 已完成的核心基线

### 工程与运行

- 前后端工程骨架已稳定
- Docker Compose 本地完整栈已可运行
- 后端、前端、基础设施已有明确分层
- 常用预检、冒烟、交付脚本已存在

### 认证与权限

- 登录、刷新、当前用户
- 邀请注册、邮箱验证、密码重置
- JWT、RBAC、多租户隔离基线

### 文档处理

- 普通上传与分片上传
- MinIO 存储
- 异步解析、切块、索引
- 状态回写、事件追踪、失败重试

### 检索与问答

- 关键词、向量、图谱、混合检索
- SSE 问答主链路
- 引用返回、会话历史、反馈提交
- Runtime V2、Replay、Resume、Checkpoint

### 管理与治理

- 系统后端状态
- 检索指标与就绪度
- 运行时任务和性能指标
- 安全审计
- 评估入口
- 企业语料导出、训练编排、模型激活入口

## 3. 最近阶段已确认的改进

以下方向已经不是停留在文档里，而是已经进入当前实现：

- 中文乱码与提示词污染修复
- 检索链路可用性和回退治理增强
- HTML 文档解析能力补强
- OCR / HTML / 检索相关回归测试补充
- 运行中的 Docker 栈已同步最新主链路改动

## 4. 当前仍需继续增强的部分

### 文档 Agent 质量

- 对追问、指代、版本歧义的处理仍需继续增强
- 专项 agent 的回答结构还可以更稳定
- “有证据但答得笨”仍然是当前核心体验问题

### 检索质量

- 当前健康度不等于真实召回质量已经足够
- 仍需持续提高命中率、去重质量、同文档聚合和弱证据治理

### 内容理解

- 高保真 OCR、复杂表格、复杂版面仍不是最终形态
- 中文长文档摘要与制度比对仍有继续优化空间

### 训练与评估

- 已有训练编排与模型激活入口
- 真实大规模训练、灰度发布、自动化评估门禁还在下一阶段

## 5. 当前阶段划分

| 模块 | 状态 | 说明 |
| --- | --- | --- |
| 工程底座 | 已完成 | 可运行、可构建、可验证 |
| 鉴权与租户 | 已完成 | 认证、RBAC、多租户已具备 |
| 文档入库 | 已完成主链路 | 复杂解析能力继续增强 |
| 检索 | 已完成主链路 | 质量仍在持续打磨 |
| Runtime | 已完成 | V2 已是主路径 |
| 前端主流程 | 已完成 | 体验细节继续优化 |
| 安全基线 | 已完成 | 具备企业内部交付基础 |
| 评估与训练编排 | 部分完成 | 框架已在，质量门禁仍待补强 |

## 6. 下一阶段执行重点

1. 持续把“能回答”提升为“回答像一个好用的文档助手”
2. 提高真实文档问答的召回和引用质量
3. 增强摘要、合规、制度理解这类高频文档任务
4. 完善评估样本、评估报告和质量门禁
5. 保持前后端中文文本、提示词和说明文档全部为 UTF-8

## 7. 验收基线

至少满足以下条件才可视为当前阶段完成：

- 后端主要测试通过
- 前端构建通过
- Docker 中前后端主链路可运行
- 上传、状态、检索、SSE、管理端入口可用
- 无新增中文乱码扩散

## 8. 当前进度判断

如果只看“系统是否可运行、是否具备文档平台主链路”，已经完成。

如果看“是否已经成为很好用的文档 Agent”，仍处于持续优化阶段，核心剩余工作集中在质量增强，而不是基础功能缺失。
