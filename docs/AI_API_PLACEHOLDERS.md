# AI API 占位边界说明

## 1. 目的

本项目允许在没有真实外部 AI 服务的情况下运行，但不允许把“假接入”和“真接入”混成一团。因此本文件用于说明哪些能力已经有真实运行入口，哪些仍然属于占位、降级或可替换实现。

## 2. 当前原则

1. 接口形态可以先稳定下来。
2. 真实 AI 不可用时，系统必须给出明确 fallback。
3. 不为了表面可用而伪造与生产形态不一致的接口契约。

## 3. 当前已有真实运行入口

- LLM 默认通过 Ollama 访问
- Embedding 默认通过 Ollama 访问
- 评估 sidecar、Guardrails sidecar、ClamAV 具备真实容器化入口
- 企业语料导出、训练任务编排、模型激活接口已存在

## 4. 当前仍属于占位或过渡实现的部分

- 某些 agent 节点仍保留规则化 fallback
- 评估在 sidecar 不可用时仍可能降级
- 训练执行器当前支持脚本编排，但不代表已经完成真实 GPU 训练生产化
- 模型发布与灰度仍未形成最终生产闭环

## 5. 开发约束

- 新增 AI 能力时，要保留明确的错误信息和降级路径
- 不要把失败静默吞掉再返回“看起来像成功”的结果
- 修改接口时要优先保持前后端契约稳定
- 中文 prompt、规则词、系统文案必须保持 UTF-8

## 6. 推荐理解方式

当前 DocMind 不是“完全脱离 AI 的假系统”，也不是“已经完成所有真实 AI 生产接入的成品”。更准确的描述是：

“已经具备真实可运行的本地 AI 接口入口，并保留部分可控 fallback，用于保证系统在开发与交付阶段稳定推进。”
